# 域名注册与备案指南

## 域名基础知识

域名是网站在互联网上的地址标识，例如 `blog.veweiyi.cn`，让用户可以通过易记的名称访问网站。

### 域名结构

```
blog.veweiyi.cn
 │    │     │
 │    │     └─ 顶级域名（TLD）
 │    └─────── 主域名
 └──────────── 子域名
```

## 域名后缀选择

### 常用后缀对比

| 后缀    | 含义    | 价格         | 适用场景  | 备案要求 |
|-------|-------|------------|-------|------|
| .com  | 商业    | 60-100元/年  | 商业网站  | 需要   |
| .cn   | 中国    | 30-50元/年   | 国内网站  | 需要   |
| .net  | 网络    | 60-80元/年   | 技术网站  | 需要   |
| .org  | 组织    | 80-100元/年  | 非营利组织 | 需要   |
| .io   | 技术社区  | 200-300元/年 | 技术博客  | 不需要  |
| .top  | 顶级    | 10-30元/年   | 个人博客  | 需要   |
| .xyz  | 新通用域名 | 10-50元/年   | 创新项目  | 需要   |
| .tech | 技术    | 100-200元/年 | 科技公司  | 不需要  |

### 选择建议

**个人博客**

- 首选：.com、.cn、.net
- 备选：.top、.xyz（价格便宜）

**企业网站**

- 首选：.com、.cn
- 同时注册多个后缀保护品牌

**技术项目**

- 首选：.io、.tech、.dev
- 体现技术属性

## 域名命名原则

### 1. 简短易记

```
✅ 好的命名
blog.veweiyi.cn      # 简短，易记
tech.example.com     # 含义明确

❌ 不好的命名
my-personal-blog-website-2024.com  # 太长
abc123xyz.com                       # 无意义
```

### 2. 避免特殊字符

```
✅ 推荐
blogsite.com         # 无连字符更好
blog-site.com        # 连字符可接受

❌ 不推荐
blog_site.com        # 下划线不支持
```

## 域名备案（国内服务器必需）

### 备案类型

| 类型   | 适用对象 | 所需材料                    | 审核时间      |
|------|------|-------------------------|-----------|
| 个人备案 | 个人   | 身份证、域名证书、承诺书            | 7-20 个工作日 |
| 企业备案 | 企业   | 营业执照、法人身份证、网站负责人身份证、授权书 | 7-20 个工作日 |

### 备案流程

```
登录云服务商备案系统 → 填写主体信息 → 填写网站信息 → 上传证件材料 → 提交初审(1-3天) → 管局审核(7-20天) → 获得备案号
```

### 备案注意事项

**重要提示**

- ⚠️ 备案期间网站不能访问
- ⚠️ 网站名称不能包含"博客"等字眼（部分地区）
- ⚠️ 备案信息必须真实有效
- ⚠️ 个人备案不能用于商业用途
- ⚠️ 备案通过前只能通过 IP 访问

**加快备案的技巧**

- ✅ 材料准备齐全，一次性提交
- ✅ 照片清晰，符合要求
- ✅ 信息填写准确，避免退回
- ✅ 及时响应服务商和管局的要求

## 域名解析配置

### DNS 记录类型

| 记录类型  | 说明         | 示例                               | 使用场景      |
|-------|------------|----------------------------------|-----------| 
| A     | 指向 IPv4 地址 | `blog.example.com → 192.168.1.1` | 最常用       |
| AAAA  | 指向 IPv6 地址 | `blog.example.com → 2001:db8::1` | IPv6 网站   |
| CNAME | 指向另一个域名    | `www.example.com → example.com`  | 域名别名      |
| MX    | 邮件服务器记录    | `example.com → mail.example.com` | 邮件服务      |
| TXT   | 文本记录       | 用于域名验证、SPF 等                     | 域名验证、安全配置 |
| NS    | 域名服务器记录    | 指定 DNS 服务器                       | DNS 托管    |
| CAA   | 证书颁发机构授权   | 限制可颁发证书的 CA                      | SSL 证书安全  |

### 解析配置示例

#### 基础配置

```
# 主域名
记录类型: A
主机记录: @
记录值: 192.168.1.1
TTL: 600

# www 子域名
记录类型: A
主机记录: www
记录值: 192.168.1.1
TTL: 600
```

#### 子域名配置

```
# 博客前台
记录类型: A
主机记录: blog
记录值: 192.168.1.1

# 管理后台
记录类型: A
主机记录: admin
记录值: 192.168.1.1

# API 接口
记录类型: A
主机记录: api
记录值: 192.168.1.1

# 静态资源
记录类型: CNAME
主机记录: static
记录值: cdn.example.com
```

#### CDN 配置

```
记录类型: CNAME
主机记录: www
记录值: www.example.com.cdn.dnsv1.com
```

```bash
nslookup blog.example.com

# dig 查询
dig blog.example.com
```

## 常见问题

### 1. 域名无法访问

**排查步骤**

```bash
# 1. 检查域名解析
nslookup blog.example.com

# 2. 检查服务器状态
ping your_server_ip

# 3. 检查防火墙
telnet your_server_ip 80

# 4. 检查 Nginx 配置
nginx -t
```

### 2. 解析不生效

**可能原因**：DNS 缓存未更新、解析记录配置错误

**解决方法**：清除本地 DNS 缓存，等待 DNS 全球同步

### 3. 备案被驳回

**常见原因**：材料不清晰、信息填写错误、网站内容不符合要求

**解决方法**：按要求补充材料，核对信息准确性
