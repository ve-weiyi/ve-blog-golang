# 分支管理和代码提交规范

## 分支管理

### 分支说明

| 分支格式                | 命名示例                   | 用途     | 稳定性    | 说明                |
|---------------------|------------------------|--------|--------|-------------------|
| `main`              | `main`                 | 主分支    | ✅ 稳定   | 最新稳定版本，保证项目可以正常运行 |
| `release/<version>` | `release/v1.0.0`       | 发布分支   | ✅ 稳定   | 记录重要版本代码          |
| `feature/<name>`    | `feature/login`        | 功能开发分支 | ⚠️ 不稳定 | 开发新功能，可能被强制推送覆盖代码 |
| `hotfix/<name>`     | `hotfix/fix-login-bug` | 热修复分支  | ⚠️ 不稳定 | 紧急修复生产环境问题        |

### 分支合并规则

```
feature/* ──→ release/* ──→ main
hotfix/* ─────────↑
    └──────────────────────→ main
```

1. 功能开发：`feature/*` → `release/*`
2. 版本发布：`release/*` → `main`
3. 紧急修复：`hotfix/*` → `main` + `release/*`

## 版本号规范

采用语义化版本号：`主版本号.次版本号.修订号`（如 `v1.2.3`）

| 版本类型 | 说明   | 示例                  | 何时升级             |
|------|------|---------------------|------------------|
| 主版本号 | 重大变更 | `v1.0.0` → `v2.0.0` | 不兼容的 API 修改、架构重构 |
| 次版本号 | 功能更新 | `v1.0.0` → `v1.1.0` | 新增功能、较大代码变动      |
| 修订号  | 问题修复 | `v1.0.0` → `v1.0.1` | Bug 修复、小优化       |

## Commit 规范

采用 [Conventional Commits](https://www.conventionalcommits.org/) 
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型

| Type       | 说明     | 示例                           |
|------------|--------|------------------------------|
| `feat`     | 新功能    | `feat: add user login`       |
| `fix`      | Bug 修复 | `fix: resolve login error`   |
| `refactor` | 代码重构   | `refactor: optimize query`   |
| `docs`     | 文档更新   | `docs: update README`        |
| `style`    | 代码格式   | `style: format code`         |
| `test`     | 测试相关   | `test: add unit tests`       |
| `perf`     | 性能优化   | `perf: optimize load speed`  |
| `chore`    | 构建/工具  | `chore: update dependencies` |
| `ci`       | CI/CD  | `ci: add test step`          |
| `build`    | 构建打包   | `build: update dockerfile`   |
| `revert`   | 回滚提交   | `revert: revert feat login`  |
| `release`  | 版本发布   | `release: bump to v1.2.0`    |
| `merge`    | 分支合并   | `merge: merge feature/user`  |
| `sync`     | 同步代码   | `sync: sync with upstream`   |

### 示例

```bash
git commit -m "feat(auth): add OAuth login support"
git commit -m "fix(api): resolve null pointer exception"
git commit -m "refactor(user): simplify profile update logic"
git commit -m "docs: update deployment guide"
```
