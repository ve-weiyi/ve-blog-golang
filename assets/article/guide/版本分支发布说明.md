# 版本分支发布说明

本文档说明 ve-blog-golang 项目的版本管理规范和发布流程。

## 分支管理

### 分支说明

| 分支名称        | 用途     | 稳定性    | 说明                          |
|-------------|--------|--------|-----------------------------|
| `main`      | 主分支    | ✅ 稳定   | 最新稳定版本，保证前后端可以正常运行          |
| `release/*` | 发布分支   | ✅ 稳定   | 记录重要版本代码，如 `release/v1.0.0` |
| `feature/*` | 功能开发分支 | ⚠️ 不稳定 | 开发新功能，可能被强制推送覆盖代码           |
| `hotfix/*`  | 热修复分支  | ✅ 稳定   | 紧急修复生产环境问题                  |

### 分支合并规则

```
feature/* → release/* → main
                ↑
            hotfix/*
```

**合并流程**：

1. 功能开发：`feature/*` → `release/*`
2. 版本发布：`release/*` → `main`
3. 紧急修复：`hotfix/*` → `main` + `release/*`

### 分支命名规范

```bash
# 功能分支
feature/user-login
feature/article-comment

# 发布分支
release/v1.0.0
release/v1.1.0

# 热修复分支
hotfix/fix-login-bug
hotfix/security-patch
```

## 版本号规范

采用语义化版本号：`主版本号.次版本号.修订号`（如 `v1.2.3`）

### 版本号说明

| 版本类型 | 说明   | 示例                  | 何时升级             |
|------|------|---------------------|------------------|
| 主版本号 | 重大变更 | `v1.0.0` → `v2.0.0` | 不兼容的 API 修改、架构重构 |
| 次版本号 | 功能更新 | `v1.0.0` → `v1.1.0` | 新增功能、较大代码变动      |
| 修订号  | 问题修复 | `v1.0.0` → `v1.0.1` | Bug 修复、小优化       |

### 版本号示例

```
v1.0.0  # 首次正式发布
v1.0.1  # 修复登录 Bug
v1.1.0  # 新增评论功能
v1.1.1  # 修复评论显示问题
v2.0.0  # 架构升级，不兼容旧版本
```

## 发布流程

### 1. 功能开发

```bash
# 创建功能分支
git checkout -b feature/new-feature main

# 开发并提交
git add .
git commit -m "feat: add new feature"

# 推送到远程
git push origin feature/new-feature
```

### 2. 版本发布

```bash
# 创建发布分支
git checkout -b release/v1.1.0 main

# 合并功能分支
git merge feature/new-feature

# 更新版本号和 CHANGELOG
vim version.go
vim CHANGELOG.md

# 提交并打标签
git commit -m "chore: release v1.1.0"
git tag -a v1.1.0 -m "Release version 1.1.0"

# 推送到远程
git push origin release/v1.1.0
git push origin v1.1.0

# 合并到主分支
git checkout main
git merge release/v1.1.0
git push origin main
```

### 3. 热修复流程

```bash
# 从主分支创建热修复分支
git checkout -b hotfix/fix-critical-bug main

# 修复并提交
git add .
git commit -m "fix: critical bug"

# 合并到主分支和发布分支
git checkout main
git merge hotfix/fix-critical-bug
git push origin main

git checkout release/v1.1.0
git merge hotfix/fix-critical-bug
git push origin release/v1.1.0
```

## Commit 规范

采用 Conventional Commits 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型

| Type       | 说明     | 示例                           |
|------------|--------|------------------------------|
| `feat`     | 新功能    | `feat: add user login`       |
| `fix`      | Bug 修复 | `fix: resolve login error`   |
| `docs`     | 文档更新   | `docs: update README`        |
| `style`    | 代码格式   | `style: format code`         |
| `refactor` | 代码重构   | `refactor: optimize query`   |
| `test`     | 测试相关   | `test: add unit tests`       |
| `chore`    | 构建/工具  | `chore: update dependencies` |

### Commit 示例

```bash
git commit -m "feat(auth): add OAuth login support"
git commit -m "fix(api): resolve null pointer exception"
git commit -m "docs: update deployment guide"
```

## 注意事项

1. ⚠️ **feature 分支不稳定**：作者可能强制推送覆盖提交，请及时同步
2. ✅ **以 main 分支为准**：生产环境部署请使用 main 或 release 分支
3. 📝 **及时更新文档**：版本发布时同步更新 CHANGELOG 和文档
4. 🏷️ **规范使用标签**：每次发布都要打 tag，便于版本追溯
